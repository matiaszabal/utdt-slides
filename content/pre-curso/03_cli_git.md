# Módulo 03 — Línea de comandos y git

## Por qué hace falta

Todos los labs se instalan y corren desde terminal (`pip install`, variables de entorno en `.env`, ejecución de scripts). El material del curso y las entregas se manejan como repositorios. No hace falta ser experto en shell scripting, pero sí moverse con soltura en una terminal y entender el flujo básico de git.

## Autodiagnóstico

1. ¿Podés navegar entre carpetas, crear archivos y borrar archivos desde la terminal sin usar el explorador de archivos gráfico?
2. ¿Sabés instalar un paquete con `pip install` y verificar que se instaló?
3. ¿Sabés qué es una variable de entorno y cómo se define en un archivo `.env`?
4. ¿Entendés la diferencia entre `git add`, `git commit` y `git push`?
5. ¿Alguna vez resolviste (aunque sea con ayuda) un conflicto de merge?

## Tiempo estimado

3-5 horas si la brecha es solo en git (la parte de terminal suele resolverse sola en la práctica). Más tiempo si nunca se usó una terminal — en ese caso, priorizar los primeros capítulos de Missing Semester.
